## 一、项目概述

Temu 卖家营销中心"批量报名活动"自动化工具。通过 Playwright 操作 Edge 浏览器，使用 browser_console + Playwright 原生模式操作 Beast UI 组件。

**核心挑战：** Beast UI 组件库（select/dropdown/modal 交互复杂）、虚拟滚动列表操作、主题交叉匹配、模板下载/上传、浏览器安全机制限制文件上传

**技术栈：** Python + Playwright（connect_over_cdp + 原生 persistent_context）

**⚠️ 全选操作的 React 更新陷阱（2026-06-12 两次踩坑被用户严厉批评）：**

全选 checkbox 位于翻页组件左侧：`.add-goods_pagination__73bvr [data-testid="beast-core-checkbox-checkIcon"]`

**核心规则：必须逐次执行，每次等 React 更新后再执行下一步。** 不能写快速循环。

✅ **正确做法（逐页手工执行，已验证 623个全部选中）：**
```

2. 第1页：点全选 → 检查"已选择(100)" → 点下一页
3. 第2页：点全选 → 检查"已选择(200)" → 点下一页
4. 重复直到下一页 disabled → 最后一页点全选 → 检查总数≈623 → 点"确认"
```

❌ **错误做法：**
- 在 browser_console 中写循环快速翻页 → React 来不及更新，click 被合并忽略
- 假设每页全选后"已选择"数字自动正确累加 → 必须观察确认

**确认按钮：** 文本是"确认"不是"确定"，`button.click()` 直接有效

**文件上传（2026-06-12 最终验证通过的方案）：**

| 方法 | 结果 | 说明 |
|:----|:----:|:-----|
| browser_console JS点击"选择文件" | ❌ | 对话框弹不出（浏览器安全限制） |
| Playwright `connect_over_cdp` + `file_chooser.set_files()` | ❌ | React 不更新 |
| Playwright 原生模式 `file_chooser.set_files()` | ✅ **稳定成功** | **需等2-3秒让 React 检测到变更** |
| 直接调上传 API | ⚠️ 可用 | 需获取 Galerie 签名，较复杂 |

**✅ Playwright 原生模式上传方法（已验证）：**

```python
from playwright.sync_api import sync_playwright
import time

USER_DATA_DIR = r'C:\Users\Administrator\AppData\Local\Microsoft\Edge\User Data'

with sync_playwright() as p:
    context = p.chromium.launch_persistent_context(
        USER_DATA_DIR,
        headless=False,
        channel='msedge',
        args=['--remote-debugging-port=9222', '--remote-allow-origins=*'],
        no_viewport=True
    )
    page = context.pages[0] if context.pages else context.new_page()
    page.goto('https://agentseller.temu.com/activity/marketing-activity')
    
    # ... 用户登录，打开Drawer，设置主题/站点 ...
    
    # 文件上传
    with page.expect_file_chooser(timeout=15000) as fc_info:
        page.locator('button:has-text("选择文件")').first.click()
    fc = fc_info.value
    fc.set_files(r'path/to/file.xlsx')
    time.sleep(3)  # 等待 React 更新显示文件名
```

⚠️ **注意事项：**
- `fi.files.length` 在 `set_files` 后立即读取仍为 0（浏览器安全限制）
- 但 **React 最终能感知到文件**，等待 2-3 秒后文件名会出现在抽屉中
- 不要用 `files.length === 0` 来判断上传是否成功，直接检查抽屉文本
- 2026-06-12 实测：2570条记录全部导入成功，0失败

1. **不得自动点击"开始导入"** — 跑到步骤⑧后，等爸爸确认价格后再上传导入

## 五、2026-06-12 实战修复记录

### 问题1：页面全面改版，旧脚本全部失效

- **现象：** 运行 `python3 报活动.py full3 6.0 20` 后，找不到"批量报名活动"按钮
- **根因：** Temu 营销活动首页全面改版
- **修复：** 停止使用旧脚本，改为新版 Drawer 流程（9步）

### 问题2：主题勾选后数量不对（前15字符匹配混淆）

- **现象：** 勾选6个主题，关闭弹窗后只显示"2 修改"
- **根因：** 两个【周末48H大折扣专区】前15字符相同
- **修复：** 改用完整名称 + `split('\n')[0]` 去除尾部标签

### 问题3：商品全选翻页太快，实际只选了543个

- **现象：** 循环翻页全选后显示543个，但应有623个
- **根因：** 快速循环中 React 来不及更新，click 被合并忽略
- **修复：** 每次只做1步（全选或翻页），下一次调用再做下一步。逐页确认"已选择(N)"数量

### 问题4：CDP 环境下所有文件上传方法都失效

- **现象：** `connect_over_cdp` 下 `set_input_files`、`file_chooser`、CDP、JS 全部无法上传
- **根因：** `connect_over_cdp` 模式下浏览器安全策略限制 file input
- **修复：** 使用 Playwright 原生模式（`launch_persistent_context`），`file_chooser.set_files()` 后等 2-3 秒让 React 更新

### 问题5：Drawer 关闭后重开导致上传失败

- **现象：** 下载模板后 Drawer 关闭，重新打开再上传过滤文件，导入无反应
- **根因：** Temu 要求整个流程在一个 Drawer 会话中完成
- **修复：** 步骤①→⑨必须在同一个 Drawer 中。意外关闭必须全部重来

### 问题6：模板文件误用旧版本

- **现象：** 用了之前生成的 (20).xlsx 而不是最新的 (24).xlsx
- **根因：** 没验证文件生成时间
- **修复：** 点击生成模板前 `date` 记时间，下载后检查文件时间戳

### 问题7：跨页面操作导致浏览器风控

- **现象：** Playwright 原生模式启动新浏览器登录 Temu 导致账号异常
- **根因：** 新浏览器新会话触发风控
- **修复：** 先 `taskkill //F //IM msedge.exe` 杀旧 Edge，再用已有 Profile + channel='msedge' 启动

## 八、v3.0 架构升级（2026-06-13）

> 升级时间: 2026-06-13
> 核心问题: 浏览器生命周期管理 + 下载事件驱动 + DOM 级数据提取

### 8.1 三层架构

```
[Workflow Layer]  报活动_全自动.py       ← 只管干活，不管理生命周期
[Runtime Layer]   hermes_browser.py     ← Edge 常驻服务，CDP 连接池
[IO Layer]        download_manager.py    ← 文件下载/上传管理
```

### 8.2 核心升级点

| 模块 | 原来 v2.x | 现在 v3.0 |
|------|-----------|-----------|
| **浏览器管理** | `with sync_playwright()` 启动子进程，Python 退出 Edge 即死 | `HermesBrowser` 常驻服务，`DETACHED_PROCESS` 独立进程，`ensure_alive()` 自动保活 |
| **活动提取** | `document.body.innerText`（全量文本，数万字符）→ Python 正则解析 | `table.TB_tableWrapper_5-120-1` DOM 级提取，JS 内完成筛选，只返回活动名数组（几十字节） |
| **下载管理** | `sleep(15)` + 轮询目录 | `DownloadManager` 事件驱动（`expect_download`）+ 文件轮询兜底，双保险，失败不崩浏览器 |
| **生命周期** | `context.close()` 5处，随时关浏览器 | 零 `context.close()`，永不开浏览器 |
| **脚本退出** | `with sync_playwright()` 退出时清理子进程 | `sync_playwright().start()` + CDP 连接，Python 退出不影响 Edge |

### 8.3 HermesBrowser 模块

**文件：** `hermes_browser.py`

```python
class HermesBrowser:
    def start_edge()      # 启动 Edge 常驻进程（DETACHED_PROCESS）
    def stop_edge()       # 手动关闭（爸爸要求时才调）
    def get_browser()     # CDP 连接（自动重连）
    def get_context()     # BrowserContext（自动恢复）
    def get_page()        # Page（自动恢复）
    def ensure_alive()    # 全链路保活：Edge → CDP → Context → Page
    def health_check()    # 返回状态字典
    def reconnect()       # 完全重连（不重启 Edge）
```

**启动方式：** `subprocess.Popen(creationflags=DETACHED_PROCESS | CREATE_NEW_PROCESS_GROUP)`

**连接方式：** `playwright.chromium.connect_over_cdp("http://127.0.0.1:9224")`

### 8.4 DownloadManager 模块

**文件：** `download_manager.py`

**双策略下载：**
1. **策略①：事件驱动** — `page.expect_download(timeout=180s)`，捕获浏览器下载事件
2. **策略②：文件轮询兜底** — 轮询 Downloads 目录，带 `seen set` 防重复检测，超时 240s

**失败处理：** 两种策略均失败 → 返回 `None`，不抛异常，不关浏览器。

### 8.5 安全性原则

1. **下载失败 ≠ 系统失败** — download 返回 None，browser 保持存活
2. **浏览器常驻** — 所有 `context.close()` 已删除，永不主动关 Edge
3. **先批准后执行** — 跑脚本前必须问爸爸
4. **步骤⑨暂停** — 核价完成后在聊天中间爸爸「继续」才导入报名

### 8.6 踩坑记录

| # | 问题 | 根因 | 解决 |
|:-:|------|------|------|
| 1 | 主题只勾了1个 | 列表侧 `innerText.trim()` 带了 `\nNew`，弹窗侧 `split('\n')[0]` 只取第一行 | 两边统一 `split('\n')[0]` |
| 2 | 浏览器「自己关闭」 | `expect_download` 超时抛异常 → `with sync_playwright()` 退出 → 子进程被 kill | 改用 `sync_playwright().start()` + CDP 连接，Edge 脱离 Python 生命周期 |
| 3 | 下载超时还在转圈 | Temu 模板生成是服务器任务（30~90秒），但 `expect_download` 默认60秒超时 | 改为180秒 + 文件轮询兜底240秒 |
| 4 | JS 正则 `\d` 不匹配 | Python `"""...\\\\d..."""` → JS `\\d` → 匹配字面反斜杠+d，不是数字 | 用 `\\d` 在 Python 源码 → `\d` 在 JS 正确 |
| 5 | `input()` 阻塞脚本 | 终端非交互模式 `input()` 抛 EOFError | 改为聊天中间爸爸，去掉 input() |
| 6 | JS `split('\n')` 断裂 | Python `"""...\n..."""` 把 `\n` 转成真实换行符 → JS 收到断行字符串 | 用 `\\n` 在 Python 源码 → `\n` 在 JS 正确 |
