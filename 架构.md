## 一、项目概述

Temu 卖家营销中心"批量报名活动"自动化工具。通过 Playwright 操作 Edge 浏览器，使用 browser_console + Playwright 原生模式操作 Beast UI 组件。

**核心挑战：** Beast UI 组件库（select/dropdown/modal 交互复杂）、虚拟滚动列表操作、主题交叉匹配、模板下载/上传、浏览器安全机制限制文件上传

**技术栈：** Python + Playwright（connect_over_cdp + 原生 persistent_context）


**文件上传方案验证：**

| 方法 | 结果 | 说明 |
|:----|:----:|:-----|
| browser_console JS点击"选择文件" | ❌ | 对话框弹不出（浏览器安全限制） |
| Playwright `connect_over_cdp` + `file_chooser.set_files()` | ❌ | React 不更新 |

### HermesBrowser 模块（v3.2.0 状态机升级）

**文件：** `hermes_browser.py`

```
class HermesBrowserV2:
    state: EDGE_OFF
           ↓
           EDGE_STARTING    # Popen(DETACHED_PROCESS) 启动 Edge
           ↓
           CDP_READY        # CDP 探活成功（/json/version → webSocketDebuggerUrl）
           ↓
           BROWSER_CONNECTED  # playwright.chromium.connect_over_cdp(9222)
           ↓
           CONTEXT_READY    # 获取/创建 browser context
           ↓
           PAGE_READY       # 获取/创建 page

    def start_edge()        # 带 CDP 前置检测：CDP 已就绪则跳过启动
    def connect_browser()   # CDP 连接（状态机推进）
    def ensure_context()    # 获取 context（状态机推进）
    def ensure_page()       # 获取 page（状态机推进）
    def get_page()          # ★ 唯一推荐入口：全链路自动推进
    def health()            # 返回 {state, cdp, browser, context, page} 快照
    def reconnect()         # 安全重连（不重启 Edge，探活后切换状态）
    def stop()              # 关闭 Edge（爸爸要求时才调）
```

**关键特性：**
- 所有操作通过 `transition()` 显式流转，不猜状态
- `_wait_cdp_ready()` 0.5s 探活循环替代固定 sleep，最长等 30s
- `_is_cdp_ready()` 检测 `webSocketDebuggerUrl` 字段而非简单 HTTP 200
- 防重复启动：`start_edge()` 先查 CDP 是否已就绪
- 向后兼容：`HermesBrowser = HermesBrowserV2` 别名，调用方零改动

**启动方式：** `subprocess.Popen(creationflags=DETACHED_PROCESS | CREATE_NEW_PROCESS_GROUP)`

**连接方式：** `playwright.chromium.connect_over_cdp("http://127.0.0.1:9222")`

### DownloadManager 模块（v2 — 文件稳定判定）

**文件：** `download_manager.py`

**架构：**
```
DownloadManagerV2
├── Event Listener（主通道）— page.expect_download()
├── File Stabilizer（核心改进）— 连续3次大小不变才确认完成
├── File Watcher（兜底）— 轮询下载目录
└── State Registry — task_id → WAITING/TRIGGERED/DOWNLOADING/DONE/FAILED
```

**下载流程：**
1. **策略①：事件驱动** — `page.expect_download(timeout=180s)`，捕获浏览器下载事件
2. **文件稳定判定**（v2 核心）— `_wait_file_stable()` 连续3次检测文件大小不变才确认完成，防 `.crdownload` 临时文件和写入未完成
3. **策略②：文件轮询兜底** — 轮询 Downloads 目录，超时 240s

**失败处理：** 返回 `None`，不抛异常，不关浏览器。

   - 已登录 `agentseller.temu.com`

### 运行命令

```bash
cd "E:/Claude code/Temu自动化/报活动" && python3 报活动_全自动.py
```

### 首次运行（需启动 Edge 常驻服务）

脚本自动执行：
1. 启动 Edge（DETACHED_PROCESS，独立进程）
2. CDP 探活（`/json/version` 0.5s 循环检测，最长 30 秒）
3. 导航到营销活动页面
4. 自动执行 9 步流程

Edge 已是常驻服务，脚本自动重连 CDP，无需等待浏览器重新加载。

### 故障排查

| 现象 | 原因 | 解决 |
|------|------|------|
| CDP 端口连不上 | Edge 还没启动或端口被占 | CDP 探活自动重试 30 秒，或 `taskkill /F /IM msedge.exe` 后重跑 |
| 找不到"批量报名活动"按钮 | 页面还没加载完或 UI 已改版 | 检查页面 URL 是否正确，人工核实 UI |
| 主题勾选数为 0 | 活动名不匹配筛选条件 | 检查页面上的活动名称和日期，或活动已过期 |
| 模板下载返回 None | 生成超时或下载失败 | 浏览器保持存活，可手动下载 |
| 核价过滤失败 | `活动核价.py` 价格不一致或文件格式变了 | 检查价格表，检查 Excel 列结构 |

### 安全红线

1. ❌ 提交无法撤回 — "开始导入"后活动即生效，用错价格会导致亏损
2. ❌ 一次 Drawer 从头到尾，不可中途关闭重开
3. ✅ 浏览器常驻，永不主动关闭 Edge
