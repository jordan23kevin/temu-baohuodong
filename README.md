## 一、项目说明

Temu 卖家营销中心"批量报名活动"自动化工具。自动完成选主题→选站点→选商品→生成模板→核价过滤→上传导入→报名的全流程。

**架构：**
- `报活动_全自动.py` — 主脚本（Workflow 层）
- `hermes_browser.py` — Edge 常驻服务（Runtime 层）
- `download_manager.py` — 下载管理器（IO 层）
- `活动核价.py` — 价格过滤工具

---

## 二、前置条件

### 2.1 系统要求
- Windows（Edge 浏览器）
- Python 3.11+
- 网络能访问 `agentseller.temu.com`

### 2.2 安装依赖

```bash
pip install playwright pandas openpyxl
playwright install msedge
```

### 2.3 浏览器准备

1. Edge 已登录 Temu（`https://agentseller.temu.com/activity/marketing-activity`）
2. Edge profile 路径：`C:\Users\Administrator\AppData\Local\Microsoft\Edge\User Data`

### 2.4 活动列表要求

页面中必须有符合条件的活动：
- 折扣 ≥ 5.0 折
- 活动天数 ≤ 20 天
- 名称不含"爆款"或"秒杀"
- 名称匹配以下白名单之一：
  - 限时6折专区（6月）
  - 周末48H大折扣专区（06/20-06/21）
  - 周末48H大折扣专区（06/27-06/28）
  - 72小时计划】夏促爆单专属链接（6.20-6.22）
  - 72小时计划】夏促爆单专属链接（6.23-6.25）
  - 72小时计划】夏促爆单专属链接（6.29-7.1）

```bash
cd "E:/Claude code/Temu自动化/报活动" && python3 报活动_全自动.py
```

脚本自动：
1. 启动 Edge 常驻进程（CDP 端口 9224）

## 四、完整流程说明

| 步骤 | 操作 | 耗时 | 说明 |
|:---:|------|:----:|------|
| ① | 分析活动列表 | ~1s | DOM 级提取，筛选≥5折/≤20天/非爆款秒杀/白名单6个 |
| ② | 打开 Drawer | ~3s | 点击「Excel批量报名活动」按钮 |
| ③ | 勾选专题活动 | ~3s | Beast UI checkIcon + 关确认弹窗 |
| ④ | 主题弹窗勾选 | ~3s | Table1 完整名称匹配，检查已选数 |
| ⑤ | 站点选择 17站 | ~5s | 下拉 108LI → 白名单勾选 → 安全检查 |
| ⑥ | 商品全选 | ~20s | 100条/页，逐页全选+确认 |
| ⑦ | 生成模板 | ~30-90s | DownloadManager 事件驱动 180s+轮询240s |
| ⑧ | 核价过滤 | ~10s | 活动核价.py，低于下限删除，高于上限降价 |
| ⑨ | 上传+导入+报名 | ~10s | file_chooser 上传 → 开始导入 → 确认报名 |
| | **总计** | **~2-3 min** | 全自动无需人工介入 |

## 六、故障排查

| 现象 | 原因 | 解决 |
|------|------|------|
| CDP 端口连不上 | Edge 还没启动或端口被占 | 等待 30 秒自动重试，或 `taskkill //F //IM msedge.exe` 后重跑 |
| 找不到"批量报名活动"按钮 | 页面还没加载完或 UI 改版 | 检查页面 URL，人工核实 UI |
| 主题勾选数为 0 | 活动名不匹配白名单 | 检查 `ALLOWED` 列表是否需更新 |
| 模板下载返回 None | 生成超时或下载失败 | 浏览器保持存活，可手动下载 |
| 核价过滤失败 | 价格表不一致或列结构变了 | 检查 `活动核价.py` 价格与 `价格表.md` 一致 |
| 上传后文件名不显示 | React 未即时刷新 | 脚本已等 3 秒，多等一会 |
| 报活动.py 运行报错 | 那是旧版脚本，UI 已下线 | 用 `报活动_全自动.py` |

## 七、安全须知

1. ❌ **提交无法撤回** — "开始导入"后活动即生效，用错价格会导致亏损
2. ❌ **一次 Drawer 从头到尾** — 不可中途关闭重开
3. ✅ **浏览器常驻** — 脚本退出不影响 Edge，后续运行自动重连
